tutorial

Everything runs locally in your browser. No files are uploaded. No login required.

Getting started

1. Drop a file onto the workbench, or click the drop zone to browse. Scry auto-detects the format and opens the right view.
2. Use the tab bar to switch between views. Tabs light up based on what your file supports (ELF binaries unlock INSPECT and DISASM; GBA ROMs unlock CART and GAME; WAV files unlock WAVE).
3. The HEX tab is always available. Click on a section, segment, or string in INSPECT and scry jumps you straight to that offset in HEX.

Supported formats

ELF RISC-V 32-bit executables, objects, shared libs. Full header, section, segment, symbol, and string inspection.
GBA Game Boy Advance ROMs (.gba). Header decode + built-in emulator powered by a modernized gbajs2 fork.
WAV RIFF/WAVE audio. Waveform visualization and playback with seek.
Raw binary Anything else opens in HEX with entropy analysis. Every byte, every time.

Gamepad

Standard gamepad layout works automatically. D-pad, face buttons (A/B), shoulder buttons (L/R), Start, and Select all map to the standard Gamepad API button indices.

V1 vs V2

Scry ships two engines. Use the mode selector in the top bar to switch:

  • V1 (PURE) — vanilla JavaScript. The original engine. Includes EMU and TRACE tabs for RV32 emulation and instruction tracing.
  • V2 (RUST/WASM) — parsing and analysis done in Rust compiled to WebAssembly. Faster, more precise, and the active development target.
EMU and TRACE are currently V1-only. V2 ports are in progress.

Privacy

Scry processes everything locally in your browser tab. No files touch a server. No data is uploaded. No login. No telemetry. Save data for the GBA emulator is stored in your browser's localStorage and never leaves your machine.
