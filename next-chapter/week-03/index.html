Week 3 -- Project Week

NCD . May 12-16, 2026 . all five days on one thing

Week 3 was project week. No lectures, no exercises, no cartridges. Just build something. I built Scry.

What is Scry

Scry is a binary workbench that runs entirely in the browser. Drop in a ROM or any binary file and it shows you what's inside: hex dump, disassembly, header breakdown, and a live emulator that actually runs the thing. No installs, no uploads, nothing leaves your tab.

The pitch is simple: take a binary and show you what it would do. The kind of tool you'd want if you were learning how programs work at the level below source code. Or if you just wanted to poke at a GBA ROM without setting up a whole dev environment.

What I built

Modes
  • Inspect -- file metadata, header fields, entry point, section table
  • Hex -- scrollable hex dump with offset markers and ASCII sidebar
  • Disasm -- ARM/Thumb disassembly with address labels and instruction highlighting
  • Emu -- live emulator (GBA via gbajs fork), runs the binary right in the page
  • Trace -- step-by-step execution trace, register state, memory reads/writes
Stack

Vanilla JS, no framework. The parent shell at /scry/ hosts tab-switched iframes (v1/v2). Design tokens in a shared tokens.css. The emulator is a forked gbajs compiled to wasm. Everything else is hand-rolled parsers and renderers.

Design

Mint-and-paper palette. Monospace everything. Light/dark mode with system preference detection. The whole thing looks like a document, not an app. Intentional: the tool should feel like reading, not like configuring.

What I learned

Binary formats are surprisingly readable once you have a hex view and a spec side by side. The GBA header is 192 bytes and tells you almost everything you need to know about what the ROM expects. Writing a disassembler from scratch is tedious but educational in the "oh, so that's what an instruction encoding actually is" way.

The hardest part was the emulator integration. Getting a gbajs fork to compile cleanly to wasm, render into a canvas inside an iframe, and respond to the parent shell's theme changes without race conditions took most of days 3 and 4.

The tutorial page and scope document came last, on day 5. Writing docs for your own tool is a different kind of thinking than building it.
