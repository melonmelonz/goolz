Verbal Mini-Demo -- The Codex CLI Session I Ran

Two minutes in the breakout. I picked Exercise 7's "persist the to-do list to localStorage" task. Goal sentence I wrote down first: "In src/TodoApp.jsx, persist todos to localStorage so it survives a page refresh; load on mount, save on every change." That was the whole scope. One file, one feature, one sentence.

The prompt referenced the file by name, mentioned the existing useState pattern, asked for a diff before applying. What came back: one useState initializer changed to read from localStorage.getItem("todos") with a JSON.parse and a try/catch, plus a useEffect that writes on every change. I read the diff and noticed the agent had also reformatted an unrelated arrow function on the way through. I rejected that line and re-prompted with "only the localStorage changes, leave the rest of the file alone." Second diff was clean. Applied.

My partner asked: "what would you do differently next session?" Two things. One, I'd add "do not modify any other code" to the original prompt instead of catching it in review. Two, I'd have asked it to add a comment explaining why the try/catch is there (corrupted localStorage), because in two weeks I won't remember. Both small. Both the kind of thing the log catches if I keep the log honest.

The Thing That Clicked Today

The .filter().filter().map() chain in Exercise 4 is the moment React stopped feeling like a UI thing and started feeling like data plumbing. The view is just a projection of an array. Whatever shape the array takes, the view follows. Filter buttons don't "change the list" -- they change what slice of the list is rendered. The data is the same data the whole time. That's the same separation I want in every system I build: the state is the truth, the view is a function of it, and the events are the only thing allowed to mutate the truth.

The Codex CLI thing that clicked: scope is the entire game. The bad-session demo in the lecture wasn't bad because the agent is dumb -- it was bad because "make this app better" forces the agent to invent the scope, and once the agent invents the scope, you're reviewing its goals against your intentions instead of the diff against the goal. Scoping the task in one sentence before the prompt is the same discipline as writing the function signature before the body. Same shape, same payoff.
