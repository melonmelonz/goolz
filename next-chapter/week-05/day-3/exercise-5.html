To-Do App

Exercise 5 / 7 . Level 2 . 45 min

The classic React to-do. Built it once with the provided starter, then reskinned the seed data as BeaglePlay TODO. Same trick as the W5D2 favorites list. Exercises that are going to live on my disk for the next week might as well track the real work. Three real tasks pre-loaded: SPI loopback verify on rev-A2, ILI9341 panel driver probe scope, miscdev framebuffer FB_FLUSH ioctl pass.

The app

import { useState } from "react";

function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: "SPI loopback verify on rev-A2",     completed: true  },
    { id: 2, text: "ILI9341 panel driver probe scope",  completed: false },
    { id: 3, text: "miscdev framebuffer FB_FLUSH ioctl", completed: false },
  ]);
  const [inputValue, setInputValue] = useState("");

  function addTodo(e) {
    e.preventDefault();
    if (inputValue.trim() === "") return;
    setTodos([...todos, { id: Date.now(), text: inputValue, completed: false }]);
    setInputValue("");
  }

  function toggleTodo(id) {
    setTodos(todos.map((t) =>
      t.id === id ? { ...t, completed: !t.completed } : t
    ));
  }

  function deleteTodo(id) {
    setTodos(todos.filter((t) => t.id !== id));
  }

  const remaining = todos.filter((t) => !t.completed).length;

  return (
    <div>
      <h1>BeaglePlay TODO</h1>
      <form onSubmit={addTodo}>
        <input
          type="text"
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
          placeholder="next bring-up step..."
        />
        <button type="submit">Add</button>
      </form>

      <ul>
        {todos.map((t) => (
          <li key={t.id}>
            <input
              type="checkbox"
              checked={t.completed}
              onChange={() => toggleTodo(t.id)}
            />
            <span style={{
              textDecoration: t.completed ? "line-through" : "none"
            }}>{t.text}</span>
            <button onClick={() => deleteTodo(t.id)}>Delete</button>
          </li>
        ))}
      </ul>

      <p>{remaining} item{remaining !== 1 ? "s" : ""} remaining</p>
      {todos.length === 0 && <p>Inbox zero. Suspicious.</p>}
    </div>
  );
}
Why [...todos, newTodo] and not todos.push(newTodo)?

React diffs state by reference, not by value. push mutates the existing array. Same reference, React sees no change, no re-render. [...todos, newTodo] builds a new array with a new reference, React sees the change, re-renders. Same lesson for toggleTodo and deleteTodo. The .map() and .filter() calls return new arrays. Immutable updates are not a style preference here. They are a correctness requirement.

Why { ...t, completed: !t.completed } in toggle?

Same reason at the object level. The other items in the list pass through by reference (untouched), but the toggled item gets a fresh object with the flipped flag. React's diff knows exactly which one changed and re-renders just that <li>. Spread, override one field, return. The pattern is identical to Rust's struct update syntax (SomeStruct { completed: !t.completed, ..t }) which is what tipped me off that React's immutable update isn't a hack. It is the same shape that languages with proper records use natively.

Why Date.now() for the id?

Quick monotonically-increasing-enough unique key for a lab. In a real app I would want a real UUID (crypto.randomUUID() is built in) because Date.now() collides if you call it twice in the same millisecond, which sounds impossible until you are hitting a button five times to test. For three keystrokes in lab it is fine. For PEEK's TUI state I would reach for the real UUID.

Extensions I added

Clear completed button. One line, setTodos(todos.filter((t) => !t.completed)). The All / Active / Completed filter buttons. Another useState for the filter mode, an extra .filter() in the chain before the map (same shape as Ex 4). Both were the kind of thing I expected to take fifteen minutes and took two. The data model was already right.

The trace

What the day brief asked for. Typing "Buy groceries": setInputValue fires on every keystroke, re-renders the input. Click Add: addTodo, preventDefault, builds new array with the new todo appended, clears input. Click the checkbox: toggleTodo(id), .map rebuilds the list with the matching item's completed flipped. Click Delete: deleteTodo(id), .filter rebuilds the list without it. At every step the state object that React stores is a fresh value. The DOM updates are a downstream consequence I never wrote code for.
