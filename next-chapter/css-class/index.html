Welcome to CSS.

HTML is the bones. CSS is the paint. Below is one plain HTML page on the left and the same page on the right — we'll get from one to the other in five tiny steps. Click Apply on each step to watch it land in the live preview.

BEFORE — index.html (no CSS)
AFTER — index.html + your CSS
1 Tell HTML where the CSS lives Applied

CSS usually lives in its own file (e.g. styles.css). You connect it to your HTML with one line inside the <head>:

<link rel="stylesheet" href="styles.css">

Then anything you write in styles.css applies to your page. As a tiny first taste of what CSS does, let's make the page readable — set a sane font and add some breathing room:

body {
  font-family: Arial, sans-serif;
  line-height: 1.6;
  padding: 20px;
  max-width: 640px;
  margin: 0 auto;
}

Hint: body means "every <body> element" — there's only one, so this styles the whole page.

2 Element selectors — paint by tag name Applied

The simplest CSS rule has a selector (who to style) and a declaration block (how to style them). Here we pick every <h1> on the page and color it:

h1 {
  color: navy;
}

h2 {
  color: teal;
  border-bottom: 2px solid teal;
  padding-bottom: 2px;
}

You can also use #some-id for one element and .some-class for groups. We'll use a class in step 5.

3 Backgrounds & colors Applied

Color isn't just for text. Anything visual has a background too. Let's give the page some warmth and turn the title into a banner:

body {
  background: #fffaf0;  /* warm off-white */
}

h1 {
  background: navy;
  color: white;
  padding: 10px 14px;
}

Colors can be names (navy), hex (#fffaf0), or rgb(). Pick what reads cleanest to you.

4 Fonts — change how text feels Applied

One font-family sets the default for everything. You can override it per element. Mixing a serif body with a sans-serif heading is a classic move:

body {
  font-family: Georgia, serif;
}

h1, h2 {
  font-family: 'Trebuchet MS', sans-serif;
  letter-spacing: 0.5px;
}

List multiple fonts in case the first one isn't installed — the browser falls back down the list.

5 Flexbox — line things up in a row Applied

By default, block elements stack vertically. To put them in a row, give their parent display: flex. Look at the three "friend" cards in the preview — they're stacked. One line of CSS lays them out side by side:

/* The three cards live inside <div class="row">…</div> */
.row {
  display: flex;
  gap: 12px;
  justify-content: center;
  flex-wrap: wrap;
}

.card {
  background: #ffeb3b;
  padding: 10px 14px;
  border: 2px solid #333;
  border-radius: 6px;
  font-weight: bold;
}

Three new ideas in one rule:

  • display: flex — children sit in a row.
  • gap — space between them (no margin math).
  • justify-content: center — pushes the whole row to the middle.

Flexbox can do way more than this — alignment, ordering, wrapping, growing, shrinking. The full guide is linked at the bottom of this page.
