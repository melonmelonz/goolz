1 Tell HTML where the CSS lives

CSS usually lives in its own file (e.g. styles.css). You connect it to your HTML with one line inside the <head>:

<link rel="stylesheet" href="styles.css">

Then anything you write in styles.css applies to your page. As a tiny first taste, let's make the page readable — set a sane font and add some breathing room:

body {
  font-family: Arial, sans-serif;
  line-height: 1.6;
  padding: 20px;
  max-width: 640px;
  margin: 0 auto;
}

Hint: body means "every <body> element" — there's only one, so this styles the whole page. margin: 0 auto is the classic centering trick.

2 Element selectors — paint by tag name

A CSS rule has two parts: a selector (who) and a declaration block (how). Here we pick every <h1> and every <h2> on the page and color them:

h1 {
  color: navy;
}

h2 {
  color: teal;
  border-bottom: 2px solid teal;
  padding-bottom: 2px;
}

Selector cheat-sheet: h1 = every tag, #some-id = the one element with that id, .some-class = every element with that class. We'll use a class in step 5.

3 Backgrounds & colors

Color isn't just for text. Anything visual has a background too. Let's give the page some warmth and turn the title into a banner:

body {
  background: #fffaf0;  /* warm off-white */
}

h1 {
  background: navy;
  color: white;
  padding: 10px 14px;
}

Colors can be names (navy), hex (#fffaf0), or rgb(). Pick what reads cleanest. Modern CSS also has hsl() and oklch() for fancier color work.

4 Fonts — change how text feels

One font-family sets the default for everything. You can override it per element. Mixing a serif body with a sans-serif heading is a classic move:

body {
  font-family: Georgia, serif;
}

h1, h2 {
  font-family: 'Trebuchet MS', sans-serif;
  letter-spacing: 0.5px;
}

List multiple fonts in case the first one isn't installed — the browser falls back down the list. End with a generic family (serif, sans-serif, monospace) so you always get something.

5 Flexbox — lay things out in a row (or column)

By default, block elements stack vertically. To put them in a row, give their parent display: flex. Look at the three "friend" cards in the preview — they're stacked. One rule lays them out side by side:

/* The cards live inside <div class="row">…</div> */
.row {
  display: flex;
  gap: 12px;
  justify-content: center;
  align-items: center;
  flex-wrap: wrap;
}

.card {
  background: #ffeb3b;
  padding: 10px 14px;
  border: 2px solid #333;
  border-radius: 6px;
  font-weight: bold;
}

Five flexbox ideas to know:

  • display: flex — children sit on the main axis (a row by default).
  • flex-direction: row | column — flip the main axis to vertical.
  • justify-content — pack items along the main axis: flex-start, center, flex-end, space-between, space-around, space-evenly.
  • align-items — line items up on the cross axis: stretch, center, flex-start, flex-end.
  • gap — space between items. Beats margin every time.

There's also flex-grow and flex-shrink on the children, for "make this one fill the leftover space" tricks. The full guide is linked at the bottom.

6 Hover & transitions — make it feel alive

CSS lets you change styles based on what the user is doing. :hover kicks in when the mouse is over an element. transition smooths the change so it doesn't jump.

.card {
  /* whenever any of these change, animate over 200ms */
  transition: background 200ms ease, box-shadow 200ms ease, transform 200ms ease;
  cursor: pointer;
}

.card:hover {
  background: #ffd24d;
  transform: translateY(-3px);
  box-shadow: 0 4px 0 rgba(0,0,0,0.25);
}

h1 {
  transition: letter-spacing 250ms ease;
}
h1:hover {
  letter-spacing: 3px;
}

After applying, hover the friend cards in the preview — they lift, the color warms, the shadow grows. Hover the <h1> too. The transition property is the magic glue.

The shape of transition: property duration timing-function delay. Common timings: ease, ease-in-out, linear, cubic-bezier(...).

7 @keyframes — full-blown animations BONUS

Transitions only animate between two states (resting → hovered). For more — looping wobbles, multi-stage fades, choreography — you write a @keyframes rule and attach it with animation.

/* 1. Define a named timeline */
@keyframes fade-in {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes wobble {
  0%   { transform: rotate(-2deg); }
  50%  { transform: rotate(2deg) translateY(-2px); }
  100% { transform: rotate(-2deg); }
}

/* 2. Attach with the animation shorthand:
   name | duration | timing | delay | iteration | direction | fill */
h1 {
  animation: fade-in 600ms ease-out both;
}

.card {
  animation: wobble 2.5s ease-in-out infinite;
}
.card:nth-child(2) { animation-delay: 0.3s; }
.card:nth-child(3) { animation-delay: 0.6s; }

/* Be a good citizen: respect users who turn motion off. */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}

What you'll see when you apply this: the title fades up into place, and the three friend cards do a staggered, gentle wobble forever — each one starts a beat after the last. That's animation-delay + :nth-child() doing the choreography.

Three concepts in one step:

  • @keyframes — a named timeline of styles, percentages from 0% → 100% (or just from/to).
  • animation shorthand — name + duration + timing + delay + how many iterations + direction + fill mode.
  • :nth-child(n) — pseudo-class that matches the Nth element. Combine with animation-delay for staggered effects.

Keep it tasteful: a 300–600ms entrance animation reads as "polished"; a constant infinite wobble on every card reads as "carnival." For UI, restraint wins.

Things to chew on · class discussion

No "right" answers; the point is to notice CSS has lots of close-but-not-equal pairs. Toss these to the room:

  • display: none vs visibility: hidden — they both make an element invisible. Are they the same? (Hint: try one and then the other on a card and watch what happens to the layout — and to keyboard tab order, and to a screen reader.)
  • px vs em vs rem vs % — four ways to say "this size." When does each make sense? (Which one breaks if a user bumps their browser font size?)
  • margin vs padding — which one collides with its neighbors and which one doesn't? (And what on earth is "margin collapse"?)
  • Why does width: 100% sometimes overflow its parent? (Three letters: b-o-x. As in box-sizing.)
  • Is !important a tool or a red flag? (Usually it means our selectors picked the wrong fight.)
  • Why doesn't text-align: center center a <div>? (What is it actually centering?)
