Fix AI's Bugs

Four small programs. Each one has exactly one bug. The rule for this exercise: I find it before I ask AI. Re-prompting until something works is the habit I'm specifically training out of.

① Program 1 — Greeting Generator

<h1 id="greeting"></h1>
<script>
  const name = prompt("What is your name?");
  const hour = new Date().getHours();
  let greeting;

  if (hour < 12) {
    greeting = "Good morning, " + name + "!";
  } else if (hour < 18) {
    greeting = "Good afternoon, " + name + "!";
  } else {
    greeting = "Good evening, " + name + "!";
  }

  document.getElementById("greetng").textContent = greeting;
</script>

Symptom: heading stays blank. Console: Cannot set properties of null (setting 'textContent').

Bug: last line — getElementById("greetng") is missing the i. The lookup returns null, then null.textContent blows up.

Fix: getElementById("greeting").

② Program 2 — Color Changer

<button onclick="changeColor()">Change Color</button>
<script>
  function changeColor() {
    const colors = ["red","blue","green","purple","orange"];
    const i = Math.floor(Math.random() * colors.length);
    const selected = colors[i];
    document.querySelector("h1").style.colour = selected;
  }
</script>

Symptom: click does nothing. Console: clean. The button isn't broken, the function ran, the color just didn't change.

Bug: style.colour. CSS-in-JS uses American spelling. Setting an unknown property on style doesn't error — JS quietly stores it as a custom property and the browser ignores it.

Fix: style.color = selected;.

Pattern: if a click "does nothing" with no error, log inside the handler first. console.log("ran") at the top of the function tells me whether the listener fired or not. From there I know which half is broken.

③ Program 3 — Counter

<h1>Count: <span id="count">0</span></h1>
<button onclick="increment()">Add 1</button>
<script>
  let count = 0;
  function increment() {
    count = count + 1;
    document.getElementById("count").textContent = "count";
  }
</script>

Symptom: first click changes the display from 0 to the literal word count and never changes again. Variable updates fine; the display doesn't reflect it.

Bug: the right-hand side is the string "count", not the variable count. Quotes turn it from a name into a value.

Fix: drop the quotes: textContent = count;.

④ Program 4 — Temperature Message

<p id="message"></p>
<script>
  const temp = prompt("Enter the temperature in Fahrenheit:");
  if (temp > 85) {
    message = "It's hot outside!";
  } else if (temp > 60) {
    message = "Nice weather!";
  } else {
    message = "It's cold! Bundle up!";
  }
  document.getElementById("message").textContent = message;
</script>

Symptom: entering 9 still says "It's hot outside!" That should be cold.

Bug: prompt() returns a string, not a number. The comparison "9" > 85 coerces both to strings (lexicographic order) because at least one side is a string in some browser paths — and even when JS coerces the other way, the literal trap is that string comparison happens whenever you don't convert the input. "9" > "85" is true: the character 9 sorts after 8.

Fix: coerce the input to a number up front.

const temp = Number(prompt("Enter the temperature in Fahrenheit:"));

Now 9 > 85 is straight number math and the branches work.

Pattern: any user input that came from prompt, input.value, URL params, localStorage — it's a string. Convert at the boundary. Once it's typed correctly, downstream code stays sane.

⑤ How to Describe a Bug to AI

Say I get stuck and AI is the right next step. The format that gets a useful answer:

I'm working on [project, one sentence].

What I expected:
  [the correct behavior, in concrete terms]

What actually happens:
  [the wrong behavior, in concrete terms]

Error message:
  [paste the full stack trace, or write "no error"]

Code:
  [the relevant section, not the whole file]

Bad vs good — same bug

BAD "my code doesn't work, help"
GOOD I'm building a tip calculator. When I enter a $50 bill and 20% tip, I expect the tip to be $10. The page shows $1000. There's no error in the console. The relevant line: const tip = bill * tipPercent;.

The "good" version contains the answer almost on its own. AI's job is to confirm and suggest bill * (tipPercent / 100). The "bad" version forces AI to guess at twelve possible problems, half of which won't apply.

PENN Same principle as prompting from Day 2: specifics in, specifics out. Vague in, generic and probably-wrong out. The bug-description format is a prompt; it's just one for a debugging context.

⑥ The "I'm Stuck" Protocol

The order I run when I hit a wall. The temptation is to skip to step 5. The skill is in steps 1 - 4.

  1. Read the error message. Actually read it.
  2. Check the basics: typos, missing semicolons, unclosed tags, wrong file path.
  3. Add console.log. Print the variables. Where does the value go wrong?
  4. Search the error string. Someone has had this exact error before.
  5. Ask AI with the structured description above.
  6. Ask a peer. Even just explaining it out loud often surfaces the answer.
  7. Ask a TA.
  8. Ask the instructor.
