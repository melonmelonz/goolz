Logic bugs don't crash. The code runs, the browser shrugs, the output is wrong. The only way through is to print every variable at every step until the numbers stop matching what I expect.

① Bug A — Average of Three Scores

This function should return the average of 80, 90, 70. The right answer is 80. The actual answer is 120. No error.

function calculateAverage(score1, score2, score3) {
  const sum = score1 + score2 + score3;
  const average = sum / 2;
  return average;
}

const myAverage = calculateAverage(80, 90, 70);
console.log("Your average score is: " + myAverage);

② Instrument the Function

Print every variable. Don't skip steps. The first place the value disagrees with my expectation is the line with the bug.

function calculateAverage(score1, score2, score3) {
  console.log("score1:", score1);
  console.log("score2:", score2);
  console.log("score3:", score3);
  const sum = score1 + score2 + score3;
  console.log("sum:", sum);
  const average = sum / 2;
  console.log("average:", average);
  return average;
}

③ The Trace

Variable Expected Actual OK?
score1 80 80 yes
score2 90 90 yes
score3 70 70 yes

average 80 120 NO

④ Diagnosis + Fix

The break is between sum and average. The line is const average = sum / 2;. Three scores divided by 2 — that's a two-score average, not three. Off-by-one in the divisor.

// before:
const average = sum / 2;

// after:
const average = sum / 3;

Or, since the function takes three explicit arguments, hard-code 3 — fine. If the function were variadic, the divisor would be scores.length.

⑤ Bug B — 10% Discount

This function applies a 10% discount to prices over $100. A $150 input should return 135. It's returning 15. No error.

function applyDiscount(price) {
  if (price > 100) {
    const discounted = price * 0.10;
    return discounted;
  }
  return price;
}

console.log(applyDiscount(150));  // expected 135 · got 15
console.log(applyDiscount(80));   // expected 80  · got 80

⑥ Trace + Diagnosis

Add a log inside the branch:

if (price > 100) {
  console.log("price:", price);          // 150
  const discounted = price * 0.10;
  console.log("discounted:", discounted); // 15  ← that's the discount AMOUNT, not the new price
  return discounted;
}

The variable is misnamed and the math is incomplete. price * 0.10 is the dollar value of the discount ($15), not the discounted price ($135). A 10% off sale means the customer pays 90% of the sticker.

⑦ The Fix

function applyDiscount(price) {
  if (price > 100) {
    return price * 0.90;     // pay 90% of original
    // or, equivalently:
    // return price - (price * 0.10);
  }
  return price;
}
PENN Logic bugs hide in the gap between what I named a variable and what it actually holds. discounted sounded right. It wasn't. The console.log caught the mismatch immediately. Don't trust the variable name; trust the number.
