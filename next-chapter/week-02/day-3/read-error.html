Read the Error

Five errors. For each one: what type, what it means in plain English, what line, what to change. The browser is not being cryptic. I just have to slow down and read.

① Anatomy of an Error Message

Every error has the same three parts. Memorize them once and the rest gets easier.

Uncaught ReferenceError: userName is not defined
    at greeting (script.js:5:22)
    at main     (script.js:12:3)
  • Uncaught ReferenceError — the type. Used a name that doesn't exist.
  • userName is not defined — the message. Tells me which name.
  • at greeting (script.js:5:22) — the where. File script.js, line 5, character 22. The lines below are the call stack — read top to bottom, that's the order things actually happened, deepest first.

② The Five Errors

A — ReferenceError on username

Uncaught ReferenceError: username is not defined
    at script.js:3:18

const userName = "Alex";
const greeting = "Hello, " + username;
console.log(greeting);
  • Type: ReferenceError.
  • Plain English: there's no variable called username.
  • Line: 3.
  • Fix: JavaScript is case-sensitive. Use the variable that exists: userName, not username.

B — TypeError on fruits.length

Uncaught TypeError: Cannot read properties of undefined (reading 'length')
    at script.js:2:24

let fruits;
console.log(fruits.length);
  • Type: TypeError.
  • Plain English: tried to read .length off fruits, but fruits has no value yet (it's undefined), and undefined has no properties.
  • Line: 2.
  • Fix: initialize the variable before reading from it: let fruits = []; (or whatever the real list is).

C — SyntaxError, extra brace

Uncaught SyntaxError: Unexpected token '}'
    at script.js:4:1

function sayHello(name) {
  console.log("Hello, " + name)
}
}
  • Type: SyntaxError.
  • Plain English: there's a closing brace with no matching opener. The grammar doesn't parse.
  • Line: 4.
  • Fix: delete the trailing }. The function already closed on line 3.

D — TypeError, capitalization

Uncaught TypeError: document.getElementByID is not a function
    at script.js:1:24

const title = document.getElementByID("main-title");
title.textContent = "Welcome!";
  • Type: TypeError.
  • Plain English: document.getElementByID isn't a thing. The real method is getElementById — lowercase d at the end.
  • Line: 1.
  • Fix: rename to document.getElementById("main-title").

E — SyntaxError, unterminated string

Uncaught SyntaxError: Unterminated string literal
    at script.js:1:20

const message = "Hello, World!;
console.log(message);
  • Type: SyntaxError.
  • Plain English: a string was opened with " but never closed before the end of the line. The parser kept reading and ran out.
  • Line: 1.
  • Fix: close the string before the semicolon: "Hello, World!".
