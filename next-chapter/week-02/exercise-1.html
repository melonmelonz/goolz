④ The Tag Table (in my own words)

Tag Purpose What It Looks Like in the Browser
<!DOCTYPE html> Tells the browser "this doc is HTML5." Not really a tag, more like a declaration at the very top. Invisible. But if you skip it, browsers fall back to "quirks mode" and stuff renders weird.
<html> The root of the document. Everything else lives inside it. Invisible — it's a wrapper.
<head> Backstage area: metadata, the title, links to CSS, etc. Not the visible page. Invisible. But its contents change the tab title, favicon, search results.
<title> Names the page. Lives inside <head>. Shows up in the browser tab and as the bookmark name.
<body> The visible page. Everything you actually see goes here. The full white page area you scroll through.
<h1> The top-level heading. Should only be one per page (the main title). Big bold text. Default browser styling makes it the largest text on the page.
<p> A paragraph — one block of body text. Normal-sized text with a little space above and below it.
<ul> "Unordered list" — a bulleted list. Order doesn't matter. An indented list with a bullet (•) before each item.
<li> One item inside a list. Lives inside <ul> or <ol>. One bulleted line.
<a> Anchor — a clickable link. The href says where it goes. Blue underlined text by default. Cursor turns into a pointer on hover.
<img> Embeds an image. src is the file, alt is the fallback text. The picture itself. If the file's missing, you get a broken-image icon plus the alt text.

⑤ Questions

Q1 — Which tag has no closing tag, and why? <img>. It's a "void" element — there's nothing to wrap because the picture is the content. The tag points at a file (src) and that's it. Same family as <br>, <hr>, <input>, <meta> — nothing goes between them, so no closer.
Q2 — What if I deleted <body> but kept its contents? Browsers are forgiving so it'd still render — modern parsers will quietly insert a <body> for you. But: any CSS targeting body stops working, JS like document.body gets weird, and you've broken the semantic structure that screen readers and search engines expect. It's the kind of thing that "works" until it really doesn't.
Q3 — Tag I had to look up <meta charset="UTF-8">. Asked: "what does <meta> do?" — <meta> is metadata about the page that lives in <head>. charset tells the browser which character encoding to decode the file with. Without it, weird stuff happens to emoji, accents, em-dashes — basically anything that isn't basic ASCII. UTF-8 covers everything.

⑥ Notes To Self

  • HTML is a tree. <html> is the root, <head> + <body> are siblings, everything else nests under one of them.
  • Comments (<!-- ... -->) don't render. Use them like inline notes.
  • "No closing tag" usually = void element. Don't try to write </img>, it's wrong.
  • Indentation isn't required, but my brain reads nested tags way faster when they line up.
