▶ EX1 / 6 — CONSOLE WARMUP

Pure-JS Drill, No Page

No HTML, no DOM, no event listeners. Just the language itself in the DevTools console. Three given functions, three of mine. The point is to feel the shape of function, parameter, return before I wrap it in any UI.

① Given

function greet(name) { return `Hello, ${name}!`; } greet("Alex"); // "Hello, Alex!" greet("Sam"); // "Hello, Sam!" function addNumbers(a, b) { return a + b; } addNumbers(3, 4); // 7 addNumbers(10, 25); // 35 function isEven(n) { return n % 2 === 0; } isEven(4); // true isEven(7); // false

② My turn

function square(n) { return n * n; } square(5); // 25 square(-3); // 9 function shout(message) { return message.toUpperCase(); } shout("hello"); // "HELLO" shout("don't yell at me"); // "DON'T YELL AT ME" function isLong(text) { return text.length > 10; } isLong("short"); // false isLong("this string is long"); // true

③ Notes

  • n * n vs Math.pow(n, 2) vs n ** 2 all work. n * n reads cleanest for the squared case. ** is what I'd reach for if the exponent were variable.
  • isLong boundary: > 10 means an 11-char string returns true, exactly 10 returns false. Spec said "more than 10", so > is right. >= would be a bug.
  • shout blows up on non-strings (shout(5) → TypeError). For console drills I'm not guarding it. In real code I'd type-check at the boundary or coerce with String(message).toUpperCase().

④ What I felt

PENN Functions are tiny machines. The console is a workbench. Type a definition, the prompt comes back. Type a call, the return value comes back. No HTML, no event glue, no surprise reload. This is the layer that was hiding under yesterday's debugging.
