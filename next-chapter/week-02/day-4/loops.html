▶ EX2 / 6 — LOOP AN ARRAY

The Same Loop, Four Ways

An array of five names. The page logs every variant to the console live. Open DevTools → Console to see four console.groups, one per step. The point: notice when I want the index and when I don't.

① The data

const names = ["Alex", "Sam", "Jordan", "Riley", "Casey"];

② Step 2 — basic for...of

Default loop for arrays. No index, no length-checking. Reads like English.

for (const name of names) { console.log(`Hello, ${name}!`); }

③ Step 3a — classic for with 1-based index

The position is part of the output, so I want the index. i + 1 because the human-readable list starts at one but the array starts at zero.

for (let i = 0; i < names.length; i++) { console.log(`${i + 1}. Hello, ${names[i]}!`); }

④ Step 3b — for...of, only names > 4 chars

The same loop, with an if inside it. continue would be equivalent for negation but reads worse here.

for (const name of names) { if (name.length > 4) { console.log(`Hello, ${name}!`); } }

⑤ Step 3c — collect into a new array

Build a parallel array, log it after. This is the shape of every transform I'll later write as .map. Same idea, hand-rolled.

const greetings = []; for (const name of names) { greetings.push(`Hello, ${name}!`); } console.log(greetings);

⑥ Step 4 — my own array

Five things I want to remember from this week.

const week2Lessons = [ "DOM glue is not the language - functions/loops/arrays/objects are", "default to const, reach for let only when reassigning", "=== always; == has weird coercion rules I do not want to memorize", "for...of for arrays unless I genuinely need the index", "data lives in an array, render the array, mutate, re-render", ]; for (const lesson of week2Lessons) { console.log(lesson); }

⑦ Lab notes

PENN for...of is the right default. The classic for earns its keep when the index is part of the output (numbering, "remove every other", walking two arrays in lockstep). If the body never reads i, for...of beats it for clarity. Save the cleverness for when it pays.
