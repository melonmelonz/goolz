▶ EXERCISE 1 / 5

Contact Form

Pure HTML, no JS. Get comfortable with the form skeleton and let the browser do the validation work for free.

① The Prompt

Build me a basic HTML contact form with name, email, and message. No CSS framework. Use semantic labels tied to inputs by id, mark required fields, and use type="email" so browsers can validate the address. No JavaScript.

② What GPT Sent Back

<form id="contact-form">
  <label for="name-input">Name</label>
  <input id="name-input" type="text" required>

  <label for="email-input">Email</label>
  <input id="email-input" type="email" required>

  <label for="message-input">Message</label>
  <textarea id="message-input" required></textarea>

  <button type="submit">Send</button>
</form>
PENN All ids match their for attributes — clicking a label focuses the input. required and type="email" are doing real work. Accepted as-is. Onto the live preview.

④ What I Tried (Lab Notes)

  • Clicked Send with everything blank → browser pops a "Please fill out this field" tooltip on the first empty input. That's required doing its job, no JS involved.
  • Typed notanemail in the email field → submit blocked, "Please include an '@' in the email address." That's type="email".
  • Clicking the label text moves the cursor into the field. Free accessibility win — comes from for= + id= matching.
  • <textarea> is NOT a void element — it has a closing tag, even when empty. <input> is the void one.

⑤ What's Missing

Right now, hitting Send doesn't actually do anything in the page — no JavaScript means no submit handler, no DOM update. That's the whole point of Exercise 2: wire up an event listener so submit means something.
