Read a Database Diagram

Exercise 1 / 5 . Level 1 . 20 min

The blog schema from the lecture. Three tables, a handful of foreign keys. The goal is to trace the connections and understand what data lives where.

The schema

USERS                    POSTS                    COMMENTS
-----------              -----------              -----------
id (PK)                  id (PK)                  id (PK)
name                     title                    body
email                    body                     created_at
created_at               published_at             post_id (FK -> posts.id)
                         user_id (FK -> users.id) user_id (FK -> users.id)

Questions

1. How many tables are in this database?

Three. users, posts, comments.

2. What is the primary key of the posts table?

id. Same as every table here. Each post gets a unique integer that never changes and never repeats.

3. How are posts connected to users? Which column creates that connection?

user_id in the posts table. It's a foreign key pointing to users.id. It says "this post was written by user #X." One user can have many posts (one-to-many).

4. How are comments connected to posts? Which column creates that connection?

post_id in the comments table. Foreign key pointing to posts.id. "This comment was written on post #X." One post can have many comments.

5. If user #2 (Student B) writes a new post titled "My Database Journey", what data gets stored in the posts table?

A new row:

Column Value
id4 (next available, assuming 3 posts already exist)
title"My Database Journey"
body(whatever Student B writes as the post content)
published_at2026-05-20 (today's date, or whenever they hit publish)
user_id2 (Student B's ID from the users table)
6. If user #1 (Student A) writes a comment on that post saying "Great post!", what gets stored in comments?

A new row:

Column Value
id1 (first comment, or whatever the next ID is)
body"Great post!"
created_at2026-05-20 (timestamp of when the comment was posted)
post_id4 (the ID of "My Database Journey" -- the post it's on)
user_id1 (Student A wrote this comment)
7. Can a post have multiple comments? How do you know?

Yes. Multiple rows in the comments table can have the same post_id. There's no rule saying a post_id can only appear once. If three people comment on post #4, there are three rows in comments all with post_id = 4. That's the "many" side of one-to-many.

8. Can a comment belong to multiple posts? How do you know?

No. Each comment row has exactly one post_id value. A single comment can't have two values in that column -- it's one cell, one value. If someone wanted to post the same text on two different posts, that would be two separate rows, two separate comments with two different IDs, that just happen to have the same body text.

What clicked

Foreign keys always live on the "many" side. Comments is the "many" side of posts-to-comments, so post_id lives in comments. Posts is the "many" side of users-to-posts, so user_id lives in posts. The table that can have many rows per parent is the one that holds the foreign key. That's the pattern.
