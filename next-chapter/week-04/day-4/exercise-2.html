SELECT Queries for ReEntry Tech

Exercise 2 / 5 . Level 1 . 25 min

All reads. Nothing changes in the database. I'm just asking questions and the database answers with rows.

The schema (reminder)

USERS (id, name, email, role, created_at, updated_at)
MODULES (id, title, description, difficulty, created_at, updated_at)
STEPS (id, module_id, title, content, step_order, created_at)
PROGRESS (id, user_id, step_id, completed, completed_at)
QUESTIONS (id, user_id, session_id, body, ai_response, created_at)
GLOSSARY_TERMS (id, term, definition, created_at, updated_at)
RESOURCES (id, title, url, description, category, created_by, created_at, updated_at)
ASSIGNMENTS (id, caseworker_id, client_id, module_id, assigned_at)

Queries

1. Get all returning citizens (not case workers) 
SELECT name, email
FROM users
WHERE role = 'citizen';

Filters on the role column. Only returns name and email because that's all the dashboard needs to show a list. No reason to pull every column.

2. Get all beginner modules, sorted by title 
SELECT title, description
FROM modules
WHERE difficulty = 'beginner'
ORDER BY title;

Two filters working together: WHERE narrows to beginner, ORDER BY sorts alphabetically. A returning citizen browsing modules would see "Setting Up Email" before "Using a Smartphone".

3. Get all steps in a specific module, in order 
SELECT title, content
FROM steps
WHERE module_id = 1
ORDER BY step_order;

This is what the app loads when someone opens a module. Steps come back in the right order because of step_order. Without ORDER BY, the database might return them in any order. I learned yesterday that order is never guaranteed unless you ask for it.

4. Get all transportation resources 
SELECT title, url, description
FROM resources
WHERE category = 'transportation';

This is exactly the query my Day 3 peer reviewer asked about. Before adding the category column (which the AI suggested), this query would have been impossible. I would've had to search inside the description text, which is unreliable.

5. Count how many questions a specific user has asked 
SELECT COUNT(*)
FROM questions
WHERE user_id = 3;

Returns a single number. Could be useful for the case worker dashboard: "this client has asked 47 questions" tells you they're engaged. Zero questions might mean they're not using the AI feature at all.

6. Get a user's completed steps for a specific module (the JOIN) 
SELECT steps.title, progress.completed_at
FROM progress
JOIN steps ON progress.step_id = steps.id
WHERE progress.user_id = 7
  AND steps.module_id = 2
  AND progress.completed = true
ORDER BY steps.step_order;

This one is harder. I need data from two tables: the step's title (from steps) and whether it's done (from progress). JOIN connects them on the foreign key. The WHERE clause filters to one user, one module, completed only. This is what the progress bar would use.

What I noticed

Every query maps directly to something in the UI. "Get all beginner modules" is the module browser page. "Get steps in order" is the module detail page. "Count questions" is a dashboard stat. SQL isn't abstract. Each query IS a feature. If I can't write the query, the feature doesn't work.
