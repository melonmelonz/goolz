SQL Syntax Crash Course

Exercise 1 / 5 . Level 1 . 20 min

Four operations, four SQL keywords. This is the entire vocabulary for today. Everything else is modifiers.

The cheat sheet

-- READ: get rows from a table
SELECT column1, column2 FROM table_name WHERE condition;

-- CREATE: add a new row
INSERT INTO table_name (column1, column2) VALUES ('value1', 'value2');

-- UPDATE: change existing rows
UPDATE table_name SET column1 = 'new_value' WHERE condition;

-- DELETE: remove rows
DELETE FROM table_name WHERE condition;

-- EXTRAS
SELECT * FROM table_name;                    -- all columns
SELECT * FROM table_name ORDER BY column;    -- sorted
SELECT * FROM table_name WHERE col = 'val';  -- filtered
SELECT COUNT(*) FROM table_name;             -- count rows

Match the operation

1. "Show me all users who are case workers"

SELECT * FROM users WHERE role = 'caseworker';

This is a READ. I'm not changing anything, just looking at rows that match a condition.

2. "A new returning citizen just signed up"

INSERT INTO users (name, email, role, created_at) VALUES ('Marcus', 'marcus@email.com', 'citizen', '2026-05-21');

This is a CREATE. A row that didn't exist before now does.

3. "The county benefits portal changed their URL"

UPDATE resources SET url = 'https://newportal.county.gov', updated_at = '2026-05-21' WHERE title = 'County Benefits Portal';

This is an UPDATE. The resource row exists, I'm just changing a column value. Also hitting that updated_at column the AI told me to add yesterday.

4. "Remove the old bus schedule link, it's been decommissioned"

DELETE FROM resources WHERE title = 'City Bus Schedule (Old)';

This is a DELETE. The row is gone. In production I'd probably soft-delete, but the SQL itself is straightforward.

Predict the output

Given this data in the modules table:

idtitledifficulty
1Setting Up Emailbeginner
2Using a Smartphonebeginner
3Online Job Applicationsintermediate
4Managing a Bank Account Onlineintermediate
Query: SELECT title FROM modules WHERE difficulty = 'beginner';

Returns two rows: "Setting Up Email" and "Using a Smartphone". The WHERE clause filters out the intermediate modules. Only the title column comes back because that's all I asked for.

Query: SELECT COUNT(*) FROM modules;

Returns 4. It counts every row regardless of difficulty. No WHERE clause means no filter.

Query: SELECT * FROM modules ORDER BY title;

Returns all four rows, all columns, sorted alphabetically by title: "Managing a Bank Account Online", "Online Job Applications", "Setting Up Email", "Using a Smartphone". ORDER BY doesn't filter, it just rearranges.

What clicked

SQL reads like English if you squint. SELECT title FROM modules WHERE difficulty = 'beginner' is basically "get the title from modules where difficulty equals beginner." The punctuation is different but the sentence structure maps almost one to one. The semicolon at the end is like a period. WHERE is the "but only if" clause.
