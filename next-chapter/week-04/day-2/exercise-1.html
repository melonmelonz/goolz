API Explorer

Exercise 1 / 5 . Level 1 . 30 min

Visit each of these public APIs in the browser. Not a webpage -- just a URL that returns raw data. That data is JSON.

API 1 -- GitHub User

https://api.github.com/users/octocat

What is the value of "login"?

"octocat"

What is the value of "public_repos"?

8

How many followers does this account have?

16212 -- this number changes because it's live data. When I checked it was over 16,000. That's the point: the data is dynamic, not baked into a page.

Try replacing octocat with your own GitHub username. Does it work?

Yes. I tried /users/melonmelonz and it came back with my account info -- my actual repos, followers, bio, everything. Same structure, different data. That's the whole idea of an endpoint: same URL pattern, different input, different output.

API 2 -- JSONPlaceholder (Fake Posts)

https://jsonplaceholder.typicode.com/posts/1

What fields do you see in the response?

userId, id, title, body. Four fields. The structure is flat -- no nested objects here.

What is the "title" of this post?

"sunt aut facere repellat provident occaecati excepturi optio reprehenderit" -- it's Latin placeholder text, like lorem ipsum but for a fake blog.

Now try /posts/2. What changed?

Different id (2), different title and body. Same structure. The userId is still 1, meaning both posts belong to the same fake user.

Now try /posts. (No number.) What do you get? How many posts are there?

An array (starts with [) of 100 post objects. So /posts = "give me all of them" and /posts/1 = "give me just this one." The number at the end is the ID.

API 3 -- JSONPlaceholder (Users)

https://jsonplaceholder.typicode.com/users/1

What is this user's name?

"Leanne Graham"

What city do they live in?

"Gwenborough" -- it's inside address.city. The address field is an object inside the user object. Objects inside objects. This is nesting.

What company do they work for?

"Romaguera-Crona" -- inside company.name. Another nested object. The company also has catchPhrase and bs fields, which is funny. Their bs is "harness real-time e-markets".

Try /users to see all users. How many are there?

10 users total. Same pattern as posts: no ID at the end returns the full list.

API 4 -- JSONPlaceholder (Todos)

https://jsonplaceholder.typicode.com/todos/1

What fields does a todo have?

userId, id, title, completed. Four fields, same as posts but with completed instead of body.

What does the "completed" field tell you?

Whether the todo is done or not. It's a boolean: true or false. Todo #1 has completed: false -- Leanne hasn't gotten around to it yet.

Visit /todos -- how many todos are there?

200. A lot of fake productivity.

Can you find a todo that's marked as completed: true?

Todo #4: "et porro tempora" with completed: true. Scrolled through the list until I found one. Roughly half are true, half are false.

What surprised me

Reflection

Honestly, how plain it is. I expected something more complicated but it's literally just text with curly braces. No colors, no layout, no buttons. Just data sitting there. Also, I didn't realize you could just type an API URL into a browser bar and get a response. I thought you had to use special tools. But the browser is just making a GET request, same as loading any page.
