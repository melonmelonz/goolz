Find a Public API

Exercise 5 / 5 . Level 3 . 25 min

The assignment: browse the public-apis list, find something relevant to ReEntry Tech, test it, and write code to call it. My app needs to explain jargon in plain language, so I looked for dictionary or definition APIs.

What I found

API info

Name: Free Dictionary API
URL: https://api.dictionaryapi.dev/api/v2/entries/en/<word>
Auth: none required (no API key)
CORS: yes (works from the browser)
Category: Dictionaries (found it under "Dictionaries" in the public-apis list)

Testing in the browser

Tried https://api.dictionaryapi.dev/api/v2/entries/en/computer and got back:

Deeply nested, but the data is there. The first definition is the archaic one (a person who computes), the second is the modern one. It also includes phonetics and source URLs. More data than I expected.

Calling it in code

// Fetch the definition of a word from the Free Dictionary API
fetch("https://api.dictionaryapi.dev/api/v2/entries/en/bandwidth")
  .then(function(response) {
    // Check if the word was found
    if (!response.ok) {
      throw new Error("Word not found (status " + response.status + ")");
    }
    return response.json();
  })
  .then(function(data) {
    // The API returns an array; grab the first entry
    var entry = data[0];
    console.log("Word: " + entry.word);

    // Loop through each meaning (noun, verb, etc.)
    entry.meanings.forEach(function(meaning) {
      console.log("\n" + meaning.partOfSpeech + ":");
      // Show the first definition for each part of speech
      console.log("  " + meaning.definitions[0].definition);
    });
  })
  .catch(function(error) {
    console.log("Error: " + error.message);
  });

Output:

Word: bandwidth

noun:
  The difference between the upper and lower frequencies
  of a range of electromagnetic frequencies.

It works. Changed the word to "firewall" and got a definition about that too. Tried a made-up word and the .catch() fired correctly with a 404.

How this fits ReEntry Tech

What data it provides

Definitions, phonetics, parts of speech, example sentences, and source URLs for English words. Covers both everyday words and technical jargon.

How it could be useful

The "explain jargon in plain language" feature from my user stories. When a returning citizen encounters a term they don't know -- "cloud," "bandwidth," "firewall," "authentication" -- the app could pull a definition from this API as a starting point. The AI chatbot could use it as a fallback or to ground its response in an actual dictionary definition before simplifying it further.

Issues

The definitions are dictionary-formal, not plain-language. "The difference between the upper and lower frequencies of a range of electromagnetic frequencies" is technically correct but useless to someone who just wants to know what "bandwidth" means in everyday terms. So the API is a data source, not a complete solution. The AI layer would still need to translate these definitions into something a human would actually say.

Also, tech terms like "two-factor authentication" aren't in the dictionary. It returns a 404 for compound phrases. So this API handles single words, not the full glossary. Good supplement, not a replacement.

Takeaway

Public APIs don't solve your whole problem. They give you a piece. The Dictionary API gives ReEntry Tech baseline definitions for single English words. The AI layer is still the thing that makes those definitions understandable. But having a real data source behind the AI (instead of it just making things up) is worth something. Grounding.
