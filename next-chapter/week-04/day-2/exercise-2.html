Fetch in Code

Exercise 2 / 5 . Level 1 . 30 min

Same thing as Exercise 1, but in JavaScript instead of the browser address bar. This is how your app will actually talk to APIs.

The base code

ChatGPT gave me this, with some tweaks to the comments:

// Make a GET request to the API
fetch("https://jsonplaceholder.typicode.com/posts/1")
  // When the response arrives, convert it from JSON to a JS object
  .then(function(response) {
    return response.json();
  })
  // When the conversion is done, use the data
  .then(function(data) {
    // Display the title in the console
    console.log("Title: " + data.title);
  })
  // If something goes wrong, show the error
  .catch(function(error) {
    console.log("Error: " + error);
  });

Saved as api-test.js, ran node api-test.js. Output:

Title: sunt aut facere repellat provident occaecati excepturi optio reprehenderit

No issues. Node 22 on this machine, so fetch is built in.

Reading the code

What does fetch() do?

Sends a request to the URL. It's the waiter walking your order to the kitchen. You hand it a URL and it goes and gets the data.

What does .then() do?

Waits for the previous step to finish, then runs a function with the result. "When the food arrives, do this with it." It's a chain -- each .then() passes its result to the next one.

What does response.json() do?

Takes the raw response (which is just text at this point) and parses it into a JavaScript object you can actually work with. Unwrapping the to-go container so you can eat the food.

What does data.title do?

Grabs the value of the "title" key from the JSON object. Dot notation -- data.title is like saying "from this data, give me the title field."

What does .catch() do?

Handles errors. If any step in the chain fails -- network down, bad URL, server error -- the chain skips to .catch() instead of crashing. "If the kitchen is on fire, tell me instead of just standing there."

Modifications

Level 1 -- display the body

fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(function(response) {
    return response.json();
  })
  .then(function(data) {
    console.log("Body: " + data.body);
  })
  .catch(function(error) {
    console.log("Error: " + error);
  });

Output: a paragraph of Latin placeholder text. Just changed data.title to data.body.

Level 2 -- fetch post #5, show title + body

fetch("https://jsonplaceholder.typicode.com/posts/5")
  .then(function(response) {
    return response.json();
  })
  .then(function(data) {
    console.log("Title: " + data.title);
    console.log("Body: " + data.body);
  })
  .catch(function(error) {
    console.log("Error: " + error);
  });

Changed the URL from /posts/1 to /posts/5 and added a second console.log. Both print.

Level 3 -- fetch all posts, show first 5 titles

fetch("https://jsonplaceholder.typicode.com/posts")
  .then(function(response) {
    return response.json();
  })
  .then(function(data) {
    // data is an array of 100 posts
    for (var i = 0; i < 5; i++) {
      console.log((i + 1) + ". " + data[i].title);
    }
  })
  .catch(function(error) {
    console.log("Error: " + error);
  });

Output:

1. sunt aut facere repellat provident occaecati excepturi optio reprehenderit
2. qui est esse
3. ea molestias quasi exercitationem repellat qui ipsa sit aut
4. eum et est occaecati
5. nesciunt quas odio

The key difference: /posts (no ID) returns an array, so data is a list. data[0] is the first post, data[0].title is its title. I used a for loop to print the first five. Could also use data.slice(0, 5).forEach() but the loop is clearer for now.
