User Stories

Who uses this, what do they need, and what does the app have to do.

The Users

Two main types: returning citizens (the people who need help) and case workers / mentors (the people supporting them). The tool needs to work for both, but the returning citizen's experience is the priority.

Stories + Features

User Story Feature
As a returning citizen, I want to ask questions about technology in plain language so I don't need to know jargon to get help. AI-powered Q&A chat (plain-language, no-jargon responses)
As a returning citizen, I want step-by-step guides for essential tasks (set up email, use a smartphone, apply for jobs online) so I can do them on my own. Guided walkthrough modules for common tasks
As a returning citizen, I want to track what I've already learned so I can pick up where I left off and see my progress. Progress tracker / completion state per module
As a returning citizen, I want explanations that don't assume I know what "cloud" or "two-factor" means so I don't feel lost or stupid. Built-in glossary / context-aware definitions (AI handles this)
As a case worker, I want to see which clients are stuck on specific steps so I know where to focus my time. Case worker dashboard with per-user progress view
As a case worker, I want to assign specific modules to clients so they have a clear starting point. Module assignment system
As a case worker, I want to add guides for local resources (city bus app, county benefits portal) so the tool is useful for our specific community. Custom guide editor / local resource library

Which stories matter most?

▶ Critical path If I remove the AI Q&A, the app is just a static tutorial site. If I remove the walkthroughs, there's no structured learning. If I remove progress tracking, people can't resume. Those three are the core. Everything else makes it better but doesn't make or break it.
